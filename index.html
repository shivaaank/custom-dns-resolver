Custom DNS Resolver

Self-hosted recursive DNS with ad/tracker blocking

Project Summary

This project implements a recursive DNS server with built-in ad and tracker blocking. It parses requests, checks against a blocklist, caches results, and resolves allowed queries upstream. Threading allows it to handle multiple queries efficiently in real-time.

System Architecture

1. Core Resolver (resolver_nometrics.cpp)

Multi-threaded UDP DNS resolver binding to 127.0.0.1:53. Implements manual DNS packet handling (QNAME encode/decode), builds and sends DNS queries to upstream/root servers, parses resource records, and constructs responses by mirroring the client's transaction ID and question and appending a single A-record answer. An in-memory DnsCache stores successful and failed lookups to reduce upstream traffic.

2. Blocklists & Keyword Tool

hosts.blocklist is a full domain hosts-format blocklist (StevenBlack). keywords.blocklist contains frequent label keywords generated by keywordgenerate.cpp, which extracts the longest label from each domain and writes frequent labels for substring/label-based blocking.

3. Request flow

  1. Startup: load `hosts.blocklist` and `keywords.blocklist` into memory.
  2. On UDP query: decode the question (QNAME/QTYPE) and normalize the domain.
  3. Block check: exact-match against the domain blocklist and substring/label match using keywords; if blocked, return NXDOMAIN or a synthetic A (commonly 0.0.0.0).
  4. Cache check: return cached success/failure immediately when present.
  5. Recursive resolution: build DNS query packets, send to configured root/upstream servers (list includes root IPs and public resolvers), follow referrals and parse answers/authority/additional sections until an A record is found or attempts/timeouts are exhausted. Upstream queries use a UDP socket with a receive timeout.
  6. Cache the result (success or failure) and respond to the client with a constructed DNS response.

4. Notes

The implementation performs low-level DNS handling without external DNS libraries. The Makefile provides helper targets to redirect system DNS to localhost on systemd-based Linux (not applicable on macOS). The repository's index.html documents usage and architecture.

Installation & Usage

  1. Enable system hook: 
    make enable
  2. Start resolver: 
    make run
  3. Restore defaults: 
    make disable
