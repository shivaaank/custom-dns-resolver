Custom Ad-Blocking Recursive DNS Resolver

A Self-Hosted Privacy Solution

Project Summary

This is a simple implementation of a custom DNS server that resolves domain names recursively. It also implements basic ad-block functionality by using a common block-list and keyword list to filter and block requests to ad servers. Threading and Caching make it efficient enough to be usable in normal traffic environments.

System Architecture

The project consists of three core components working in unison:

1. The Core Resolver (resolver_nometrics.cpp)

A multi-threaded UDP server listening on port 53. It parses incoming packets, checks against an in-memory set of blocked domains, and resolves allowed queries upstream.

2. The Controller (Makefile)

Automates the complex task of unbinding the OS's default DNS stub listener. It safely backs up /etc/resolv.conf before injecting the localhost address.

3. The Blocklist (hosts.blocklist)

A curated list of malicious and commercial domains (https://raw.githubusercontent.com/stevenblack/hosts/master/hosts), parsed at runtime for efficient lookups.

Installation & Usage

To deploy this resolver on a standard Linux environment:

  1. Enable the System Hook:
    Stops the default resolver and redirects system DNS to localhost. 
    make enable
  2. Start the Resolver:
    The server will now filter traffic in real-time. Logs will appear for [QUERY] and [BLOCKED] events. 
    make run
  3. Restore Default Settings:
    Reverts system changes and restores the original DNS configuration. 
    make disable
