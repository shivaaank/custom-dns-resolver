Custom DNS Resolver

Self-hosted recursive DNS with ad/tracker blocking

Project Summary

This project implements a recursive DNS server with built-in ad and tracker blocking. It parses requests, checks against a blocklist, caches results, and resolves allowed queries upstream. Threading allows it to handle multiple queries efficiently in real-time.

System Architecture

1. Core Resolver (resolver_nometrics.cpp)

Multi-threaded UDP server that parses queries, checks blocked domains, and resolves allowed domains recursively.

2. Controller (Makefile)

Automates redirecting system DNS to localhost safely and restores the original configuration on exit.

3. Blocklist (hosts.blocklist)

Curated list of ad and tracker domains (StevenBlack/hosts) parsed at runtime for efficient lookups.

Installation & Usage

  1. Enable system hook: 
    make enable
  2. Start resolver: 
    make run
  3. Restore defaults: 
    make disable
